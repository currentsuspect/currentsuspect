## Hyprland Desktop Ricing

### nwg-dock-hyprland Autohide Fix
- Bottom dock (`nwg-dock-hyprland`) wasn't showing on hover
- Fixed by changing `-hd 64` (hotspot delay) to `-hd 0` (no delay) in `hyprland.conf`
- Also set `-hl overlay` to ensure hotspot layer isn't blocked
- Config line: `exec-once = nwg-dock-hyprland -d -i 48 -mb 10 -hd 0 -hl overlay`
- Config file: `~/.config/hypr/hyprland.conf`

### Bootloader Migration (systemd-boot → rEFInd → GRUB)
- Replaced systemd-boot with macOS-style boot picker
- First tried **rEFInd** — installed and configured with dark theme + macOS blue selection highlights
- Problem: HP firmware ignores UEFI boot order, prefers legacy BBS entry (`Boot0000* Notebook mSATA Drive`)
- rEFInd never showed — firmware bypassed it
- Removed rEFInd, switched to **GRUB** with WhiteSur theme (macOS Big Sur style)
- Key fix: deleted the legacy BBS entry via `efibootmgr -b 0000 -B` so firmware can't skip UEFI entries
- Also replaced fallback bootloader (`/boot/EFI/BOOT/BOOTX64.EFI`) with GRUB as extra safety net
- GRUB config: `GRUB_THEME="/usr/share/grub/themes/whitesur/theme.txt"`, timeout 3s, 1080p
- Boot order: `Boot0005* GRUB` first, `Boot0004* Linux Boot Manager` (systemd-boot) as fallback
- User's HP firmware (UEFI 2.10, HPQ 4096.01) has quirky boot behavior — doesn't respect `BootOrder` when BBS entries exist

### Boot Order Issue (HP Firmware)
- `efibootmgr` shows "No BootOrder is set" after certain operations — firmware resets boot order
- HP firmware at UEFI 2.10 prefers BBS entries over UEFI entries
- Workaround: delete BBS entries that conflict, set `BootNext` for one-time forced boot
