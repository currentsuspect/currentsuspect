# 2026-03-19 — Construction Records App Development

## Morning Session (08:00–09:30 UTC)

**Project:** Construction Records Management System for Kat  
**Stack:** Next.js 14, Prisma, Neon PostgreSQL, Tailwind CSS, Vercel

### What We Built

1. **Dark Mode Implementation**
   - Added `next-themes` integration
   - Theme toggle button in header (🌙/☀️)
   - Full dark mode styles across all components
   - System preference detection

2. **Database Migration (SQLite → PostgreSQL)**
   - Migrated to Neon PostgreSQL for Vercel persistence
   - SQLite doesn't persist on serverless (filesystem is ephemeral)
   - Set up `DATABASE_URL` env var in Vercel
   - Ran Prisma migrations on Neon
   - Seeded initial data

3. **Full CRUD Operations**
   - **Projects:** Edit (name, budget, status, progress, dates), Delete with confirmation
   - **Clients:** Edit (name, email, phone, address), Delete with confirmation  
   - **Expenses:** Edit (category, description, amount, date), Delete with confirmation
   - **Invoices:** Delete with confirmation (edit intentionally omitted for audit trail)

4. **API Routes Created**
   - `PATCH /api/projects/[id]` — Update project
   - `DELETE /api/projects/[id]` — Delete project
   - `PATCH /api/clients/[id]` — Update client
   - `DELETE /api/clients/[id]` — Delete client
   - `PATCH /api/expenses/[id]` — Update expense
   - `DELETE /api/expenses/[id]` — Delete expense
   - `DELETE /api/invoices/[id]` — Delete invoice

5. **UI Improvements**
   - Edit/delete buttons on all cards and tables
   - Modal forms for editing with pre-populated data
   - Confirmation dialog before deletion
   - Dark mode support for all modals and forms
