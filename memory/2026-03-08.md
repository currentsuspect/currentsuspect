### Critical Thread Safety Fixes Applied (15:18-16:09 UTC)
- **Fix 1**: CommandHistory callback deadlock
  - Release lock before calling `m_onStateChanged` callback
  - Prevents deadlock if callback queries history state
  - Applied to `pushAndExecute()`, `undo()`, `redo()`, `clear()`
- **Fix 2**: PlaylistModel reader-writer mutex
  - Changed `std::mutex` to `std::shared_mutex`
  - Audio thread (reader) uses `std::shared_lock`
  - UI thread (writer) uses `std::unique_lock`
  - Prevents audio glitches from priority inversion
- **Fix 3**: ICommand::redo() pure virtual
  - Forces explicit redo() implementation for all commands
  - Added redo() to FailingCommand test helper
- **Bonus**: Optimized trimHistory() to O(n) batch erase
- All 14 tests passing. Committed: `a610f82`
