## Aestra Bug Hunting Session

### Bug 1: Clips not rendering on timeline
**Root cause:** `PlaylistModel::addClipFromPattern()` stored pattern ID in `clip.sourceId` but never set `clip.patternId`. All rendering code (`drawClipAtPosition`, `drawSampleClipForClip`, `drawWaveformForClip`) reads `clip.patternId` to resolve pattern → audio source → waveform.
**Fix:** Added `clip.patternId = patternId;` in `PlaylistModel.h:addClipFromPattern()`
**File:** `/home/currentsuspect/Aestra/AestraAudio/include/Models/PlaylistModel.h`

### Bug 2: Timeline audio not playing
**Root cause:** `PlaylistModel::buildRuntimeSnapshot()` did `sources.getSource(ClipSourceID{clip.sourceId})` — but `sourceId` held the **pattern ID**, not the source ID. Pattern IDs and source IDs have separate counters (PatternManager vs SourceManager), so lookup always returned null → no audio buffer → silent playback.
**Fix:** Changed `buildRuntimeSnapshot` to resolve through chain: `clip.patternId` → `PatternSource` → `AudioSlicePayload::audioSourceId` → `ClipSource`. Also properly marks MIDI clips as `isAudioClip = false`.
**File:** `/home/currentsuspect/Aestra/AestraAudio/include/Models/PlaylistModel.h`

### Bug 3: Linux audio drivers not registering (still in progress)
**Root cause:** `DriverRegistryStub.cpp` provides a weak symbol for `RegisterPlatformDrivers`. `LinuxDriverRegistry.cpp` provides the strong symbol. But link order puts `libAestraAudioLinux.a` BEFORE `libAestraAudioCore.a` in the link command. GNU ld processes left-to-right: when it hits `AudioDeviceManager.cpp.o` (in Core) which needs `RegisterPlatformDrivers`, it already passed the Linux lib and won't go back. The weak stub satisfies the reference instead of the strong Linux implementation.
**Fix in progress:** 
1. Commented out `DriverRegistryStub.cpp` from AestraAudioCore sources on Linux
2. Changed `AestraAudioLinux` to link `AestraAudioCore` as PRIVATE (not PUBLIC)
3. Added explicit `AestraAudioCore` after `AestraAudio` in `Source/CMakeLists.txt` link order
**Files:** `AestraAudio/CMakeLists.txt`, `Source/CMakeLists.txt`
**Status:** Build failing — need to reconfigure cmake and rebuild
