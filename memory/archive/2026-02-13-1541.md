# 2) Training & Loss (Core Mechanics)

## 1) What training really is
A model is trained to **predict the next token**.  
That’s it. Everything else emerges from that.

Given text:
```
“Kat is my ____”
```
The model is trained to output: `world`, `heart`, etc.

It sees billions of examples like this.

---

## 2) Objective function (loss)
The model outputs a probability distribution over the vocabulary.  
We compare it to the correct token and compute **cross‑entropy loss**:

\[
\mathcal{L} = -\log P(\text{correct token})
\]

So if the model assigns low probability to the correct token → loss is high.  
Training tries to **minimize this loss**.

---

## 3) Gradient descent (the engine)
Training uses **backpropagation** to compute gradients of loss w.r.t. every parameter.  
Then we **update weights**:

\[
\theta \leftarrow \theta - \eta \nabla_\theta \mathcal{L}
\]

Where:
- \(\eta\) is learning rate (step size)
- \(\nabla_\theta\) is the gradient

---

## 4) Why it generalizes
The model doesn’t memorize every sentence — it learns patterns:
- syntax
- word relationships
- structure
- long‑range dependencies

Large models generalize better because they can represent more structure.

---

## 5) Overfitting vs underfitting
- **Underfitting:** model is too weak → high loss everywhere  
- **Overfitting:** model memorizes → low training loss, bad real‑world performance  

We fight overfitting with:
- dropout
- data diversity
- regularization
- early stopping

---

## 6) Scaling laws (why size matters)
Empirically:
- More data + more parameters + more compute ⇒ lower loss
- But with **diminishing returns**
- Scaling laws help us predict performance as we scale

---

## 7) Why loss ≠ “intelligence”
Loss measures **prediction accuracy**, not truth or reasoning.
That’s why models can:
- sound right but be wrong
- be confident and still hallucinate

assistant: Post‑training is what happens **after the base model is trained on next‑token prediction**. It’s how we shape behavior, safety, and usefulness.

### 1) **SFT (Supervised Fine‑Tuning)**
Humans write ideal answers to prompts.  
Model is trained to imitate those responses.  
→ Teaches helpfulness, style, task structure.

### 2) **Preference Training (RLHF / DPO / PPO)**
Humans compare two outputs: “A vs B”.  
Model learns a **reward model** or directly optimizes preferences.  
→ Reduces toxic/unsafe output, improves instruction‑following.

### 3) **Safety Layering**
Extra filters/rules enforce policy.  
→ Blocks disallowed content even if the base model would answer.

### 4) **System Prompts + Policies**
At runtime, the model is given a **hidden policy prompt**.  
→ Steers behavior without changing weights.

### 5) **Inference/Decoding**
Model uses sampling rules to generate text:
- **temperature** (creativity)
- **top‑p/top‑k** (limit token choices)
- repetition penalties, etc.

### 6) **Tools/RAG (optional)**
Model calls tools (search, code, DB) or retrieves from a knowledge base.  
→ Adds fresh info and reduces hallucinations.

---

So “post‑training” is basically:  
**(base model) + (instruction tuning) + (preference shaping) + (runtime policy + decoding).**
