## Why Vercel is Tricky for This

Your Blueprint app has **local dependencies**:
- SQLite database (lives on the VPS)
- Challenge files (`.secret_message.txt`, `forbidden.txt`, etc.)
- Vault synced to Obsidian (`~/Everything/`)

Vercel is **serverless** — each request spins up a fresh container that dies after. No persistent filesystem. Database writes would be lost.

---

## Better Options

### 1. **Cloudflare Tunnel** (Easiest — you already have it!)
Your VPS already has `cloudflared` running. We just add a route for port 8090:
```
blueprint.yourdomain.com → localhost:8090
```

### 2. **Vercel + External Database** (Possible but more work)
- Move SQLite → Supabase/PlanetScale/Turso
- Move files → S3/R2
- Rewrite the app to be stateless

### 3. **Railway / Render / Fly.io** (Good for FastAPI)
These support persistent volumes and long-running processes.
